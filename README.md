# Chicago Rent Alpha Predictor

End-to-end CMSE 492 final project exploring what drives **ZIP-level rent outperformance**
("alpha") in the Chicago metro area.

The goal is to:
- Engineer economic, structural, lifestyle, and safety features at the ZIP Ã— month level
- Train and compare linear vs tree-based models (Ridge, Gradient Boosting, Random Forest)
- Identify which neighborhoods are projected to outgrow the metro baseline, and why

How to Run the Project
This project contains two primary Jupyter notebooks that should be executed in order.

1. Setup Environment (Recommended)
Create and activate a virtual environment:
python -m venv .venv

Activate the environment:

macOS / Linux:

source .venv/bin/activate


Windows:

.\.venv\Scripts\activate

Install dependencies:
pip install -r requirements.txt

(Optional) Register the kernel so it appears in Jupyter:
python -m ipykernel install --user --name cmse492_project --display-name "Python (cmse492_project)"

Launch Jupyter:
python -m notebook

Data Sources

This project integrates multiple public datasets related to housing, crime, demographics, and development activity in Chicago:

ğŸ”¹ Zillow Observed Rent Index (ZORI)

Source: Zillow Research
URL: https://www.zillow.com/research/data/

Provides monthly ZIP-level rent estimates for the Chicago metropolitan area.

Used to compute the target variable: 12-month forward rent growth relative to the metro average (â€œrent alphaâ€).

ğŸ”¹ Chicago Crime Data

Source: City of Chicago Open Data Portal â€” Crimes (2001â€“Present)
URL: https://data.cityofchicago.org

Includes geocoded crime reports generated by the Chicago Police Department.

Monthly counts were aggregated and smoothed using 12-month rolling sums.

ğŸ”¹ Chicago Building Permit Data

Source: Chicago Data Portal â€” Building Permits
URL: https://data.cityofchicago.org

Records new construction and renovation permits.

Used as a proxy for neighborhood construction activity and redevelopment intensity.

ğŸ”¹ Demographic and Socioeconomic Indicators

Source: U.S. Census Bureau (ACS 5-year estimates)

Provides ZIP-level income, poverty, vacancy rate, and housing age.

Used to engineer structural features and amenity interactions.

ğŸ”¹ Engineered Lifestyle & Amenity Proxies

Because APIs such as Yelp could not be used within project constraints, lifestyle indicators were approximated using:

Housing age

Transit access measures

Nightlife intensity (constructed using demographic + spatial heuristics)

Hipster index and wellness index (engineered features)

These proxies were validated through exploratory correlation and feature importance analysis.
